# 주식 갭&눌림목 봇 — 한국투자증권 KOSPI/KOSDAQ

> **대상 독자:** 봇을 운영하거나 코드를 유지보수하는 사람  
> **목적:** "봇이 왜 이렇게 작동하는가"를 이해하기 위한 레퍼런스

---

## 1. 전략 개요

### 갭&눌림목(Gap & Pullback)이란?

주식이 전날 종가보다 **높게 시작(갭 업)** 한 뒤 잠시 조정을 받다가 **다시 반등**하는 순간을 노리는 전략이다.

```
가격
  ↑
  │              ┌─── TP3 목표
  │         ┌───┤
  │    ← 고점 형성 후
  │   ┌─┘   하락(눌림목)
  │  ┌┘
갭 업│         └┐  ← 반등 확인 → 진입!
  ─ ┘           └──────────────────────────
  │
전일 종가 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
```

**직관적 설명:**
1. 좋은 소식으로 주가가 갭 업 시작 → 초반 강세 종목 선별
2. 초기 모멘텀에 매수한 사람들이 차익 실현 → 일시적 하락(눌림목)
3. 하락이 과도하지 않으면 지지선에서 반등 → **이 순간이 진입 타이밍**
4. 갭 방향(상승)으로 다시 움직임 → 단기 차익 실현

### 왜 09:20~11:20인가?

```
09:00~09:20: 장 초반 혼조 구간
  - 호가 스프레드가 넓음
  - 체결강도 집계가 불안정 (KIS API 초기값 0 반환)
  - 갭이 실제 관심인지 알 수 없음

09:20~11:20: 봇의 거래 시간
  - 체결강도 집계 안정화 완료
  - 갭이 장 시작 20분 후에도 유지되면 관심 종목으로 확인됨
  - 오전 장 유동성이 가장 높은 시간대

11:20 이후: 강제 청산
  - 오후로 갈수록 유동성 감소, 변동성 증가
  - 당일 수익을 확정하고 익일 리스크 없음
```

### 잘 작동하는 조건

- 갭 상승이 2~7% 사이 (너무 크면 이미 과열, 너무 작으면 모멘텀 부족)
- 체결강도 100 이상 (매수 > 매도)
- 시가총액 500억 이상 (유동성 확보)
- 눌림목이 -1.5% ~ -3% 범위 (너무 얕으면 눌림목이 아님, 너무 깊으면 하락 추세)

### 불리한 조건

- 공휴일 (봇은 현재 평일 스케줄만 있음, 공휴일 필터 미구현)
- 시장 급락 구간 (손절 -1.5%가 자주 발동)
- KIS API 점검 시간

## 3. 종목 스크리닝

### 관심 종목 풀

약 60개의 종목을 대상으로 매일 스크리닝을 수행한다:
- 반도체 대형주 (삼성전자, SK하이닉스 등)
- 배터리/EV (LG에너지솔루션, 삼성SDI, 에코프로비엠 등)
- 바이오 (삼성바이오로직스, 에이비엘바이오 등)
- 테크/게임 (카카오게임즈, 크래프톤, 펄어비스 등)

### 1단계: Floor 필터 (Hard Cut)

다음 조건 중 하나라도 미충족이면 후보에서 제외:

```
① 시가 데이터 존재 여부: 시가가 0이거나 없으면 제외 (장 초반 데이터 지연)

② 갭 범위: 0.5% ~ 15%
   - 0.5% 미만: 갭 아님, 모멘텀 없음
   - 15% 초과: 상한가 근처, 진입 불가

③ 체결강도: 95 이상 (총 매수체결량 / 총 매도체결량 × 100)
   - 95 미만: 매수보다 매도가 많음 → 제외
   - 단, 0인 경우(KIS API 집계 미완료)는 통과 허용

④ 시가총액: 500억 이상
   - 미만이면 소형주, 슬리피지/유동성 리스크 높음
```

### 2단계: 복합 점수 계산 (5가지 요소)

Floor 필터를 통과한 종목에 대해 0~100점 점수를 계산한다:

```
┌────────────┬──────┬────────────────────────────────────────────────────┐
│ 요소       │ 가중치│ 점수화 방식                                       │
├────────────┼──────┼────────────────────────────────────────────────────┤
│ 갭         │  30% │ 벨 커브: 4% 중심, σ=3%                           │
│            │      │ → 3~5% 갭이 최고점, 0%나 15%는 0점              │
│ 체결강도   │  25% │ 선형: 95~130 범위 정규화                          │
│            │      │ → 130 이상 = 100점, 95 = 0점                    │
│ 거래대금   │  20% │ 로그 스케일: 5억~500억 범위                       │
│            │      │ → 유동성 지표, 로그 스케일로 극단값 완화          │
│ 스프레드   │  15% │ 역수: 0~0.5% 범위 (작을수록 좋음)                │
│            │      │ → 매도호가-매수호가 차이가 작으면 고점수          │
│ 시가총액   │  10% │ 로그 스케일: 500억~10조 범위                      │
│            │      │ → 안정성 지표, 소형주 패널티                      │
└────────────┴──────┴────────────────────────────────────────────────────┘
```

### 갭 점수 벨 커브 — 왜 4%가 중심인가?

```
갭 점수
 100% │              ★ 최고점 (4% 갭)
      │           ╱     ╲
  75% │        ╱           ╲
      │     ╱                 ╲
  50% │  ╱                       ╲
      │╱                             ╲
   0% ├──────────────────────────────────── 갭%
      0%  2%  4%  6%  8%  10%  12%  15%

공식: 점수 = e^(-(갭-4)² / (2×3²))

이유:
- 2% 미만: 갭이 미미해서 초기 모멘텀 부족
- 3~5%: 충분한 갭이면서 과열되지 않은 "황금 구간"
- 7% 초과: 이미 급등, 눌림목이 너무 깊어질 가능성
- 15%: 상한가 근처, 추가 상승 여지 제한
```

### 3단계: 종목 선정

```
정렬: 복합 점수 내림차순
선정: 상위 N개 (최대 watchlistSize, 기본 10개)
보장: 점수 하한(40점) 미달이더라도 minCandidates(3개)는 보장
```

## 4. 8단계 상태 머신 ★ 핵심

각 종목은 아래 상태를 순서대로 거쳐야만 진입 가능하다.

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  [WATCHING]                                                          │
│       │                                                              │
│       │ 현재가 ≥ 시가 × 1.015  (+1.5% 이상 상승)                  │
│       ▼                                                              │
│  [HIGH_FORMED]  ← 초기 강세 확인                                    │
│       │                                                              │
│       │ 현재가 ≤ 고점 × 0.985  (고점 대비 -1.5% 이상 하락)         │
│       │                                                              │
│       │                    고점 대비 -3% 초과 하락이면              │
│       │                              ↓                              │
│       │                    [FILTERED_OUT] ← 이탈, 더 이상 추적 안 함│
│       ▼                                                              │
│  [PULLBACK]  ← 눌림목 확인                                          │
│       │                                                              │
│       │ 현재가 ≥ 눌림목 저점 × 1.003  (저점 대비 +0.3% 반등)       │
│       │ + 진입 검증 (체결강도, 호가 불균형, 눌림목 시간)            │
│       ▼                                                              │
│  [ENTRY_READY]  ← 반등 확인, 매수 대기                              │
│       │                                                              │
│       │ 매수 주문 체결                                               │
│       ▼                                                              │
│  [ENTERED]  ← 포지션 보유 중                                        │
│       │                                                              │
│       │ 손절 / TP1 / TP2 / TP3 / 추적손절 / 시간청산               │
│       ▼                                                              │
│  [EXITED]  ← 청산 완료                                              │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 각 상태 전환 조건 상세

| 전환 | 조건 | 이유 |
|------|------|------|
| WATCHING → HIGH_FORMED | 현재가 ≥ 시가 × 1.015 | 갭 업 이후 최소 +1.5% 추가 상승이 있어야 실제 모멘텀으로 인정 |
| HIGH_FORMED → PULLBACK | 현재가 ≤ 고점 × 0.985 | 고점에서 -1.5% 이상 하락 시 눌림목 구간으로 진입 |
| PULLBACK → FILTERED_OUT | 현재가 < 고점 × 0.970 | -3% 초과 하락 → 단순 눌림목이 아니라 하락 추세로 판단 |
| PULLBACK → ENTRY_READY | 저점 × 1.003 이상 반등 + 진입 검증 통과 | 반등 확인 + 매수 압력 확인 |
| ENTRY_READY → ENTERED | 매수 주문 체결 | 실제 포지션 진입 |
| ENTERED → EXITED | 청산 조건 (섹션 6) | 수익 실현 또는 손실 제한 |

## 5. 진입 조건 상세

ENTRY_READY 전환 시 3가지 조건을 검증한다.

### 소프트 검증 (softEntryValidation = true, 기본값)

**3개 중 2개 이상 충족**하면 진입 허용 (2/3 규칙)

```
조건 1: 체결강도 ≥ 100
  - 공식: 총 매수체결량 / 총 매도체결량 × 100
  - 의미: 매수 압력이 매도 압력과 같거나 큼
  - 미충족: 아직 매도세가 강함

조건 2: 호가 불균형(Order Imbalance) ≥ 1.0
  - 공식: 총 매수호가 / 총 매도호가
  - 의미: 매수 대기 주문이 매도 대기 주문보다 많음
  - 미충족: 매도 압력이 더 강함

조건 3: 눌림목 지속 시간 3~15분
  - 3분 미만: 너무 빠른 반등 → 가짜 반등(휩소) 가능성
  - 15분 초과: 오래된 눌림목 → 패턴이 식었음
```

### 하드 검증 (softEntryValidation = false)

3개 조건 모두 충족해야 진입 허용 (더 보수적)

## 6. 청산 전략 — 3단계 익절

### 청산 우선순위 (높은 순서대로)

```
┌────────────┬──────────────────────────────┬──────────────┬───────────┐
│ 구분       │ 조건                         │ 청산 수량    │ 우선순위  │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ 손절       │ 현재가 ≤ 진입가 × 0.985      │ 잔여 100%    │ 최우선    │
│            │ (-1.5%)                      │              │           │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ TP1        │ 현재가 ≥ 진입가 × 1.015      │ 진입수량 50% │ 2순위     │
│            │ (+1.5%)                      │              │           │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ TP2        │ 현재가 ≥ 당일 고점           │ 잔여의 60%   │ 3순위     │
│            │ (TP1 이후 가능)              │              │           │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ TP3        │ 현재가 ≥ 당일 고점 × 1.01   │ 잔여 100%    │ 4순위     │
│            │ (고점 대비 +1%)              │              │           │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ 추적손절   │ 현재가 ≤ 추적고점 × 0.992   │ 잔여 100%    │ 5순위     │
│            │ (-0.8%) [TP1 이후 활성화]    │              │           │
├────────────┼──────────────────────────────┼──────────────┼───────────┤
│ 시간청산   │ 11:20 이후                   │ 잔여 100%    │ 최후 수단 │
└────────────┴──────────────────────────────┴──────────────┴───────────┘
```

### 시나리오 예시

```
진입가: 10,000원 / 100주 매수 (진입금액: 1,000,000원)

─────────────────────────────────────────────────────────────────────────
시나리오 1: 상승 지속 (모든 TP 달성)

  10,150원 (+1.5%) 도달 → TP1 실행
    → 50주 매도, 잔여 50주
    → 추적손절 활성화 (추적고점 = 10,150원, 추적손절 = 10,150 × 0.992 = 10,069원)

  10,200원 (당일 고점 갱신) 도달 → TP2 실행
    → 50주 × 60% = 30주 매도, 잔여 20주
    → 추적고점 = 10,200원, 추적손절 = 10,200 × 0.992 = 10,118원

  10,302원 (10,200 × 1.01) 도달 → TP3 실행
    → 잔여 20주 전량 매도

  결과: 100주 전량 청산, 각 구간 수익 실현

─────────────────────────────────────────────────────────────────────────
시나리오 2: TP1 이후 하락 (추적손절)

  10,150원 → TP1 실행 (50주 매도, 잔여 50주)
  10,300원 → 최고가 갱신, 추적손절 = 10,300 × 0.992 = 10,218원
  가격 하락 → 10,218원 도달 → 추적손절 발동
    → 잔여 50주 전량 청산

  결과: TP1 수익 + 추적손절 수익 (약 +2.18%)

─────────────────────────────────────────────────────────────────────────
시나리오 3: 즉시 하락 (손절)

  진입 후 하락 → 9,850원 (진입가 × 0.985) 도달
    → 잔여 100% 즉시 청산 (손절)

  결과: -1.5% 손실 확정
─────────────────────────────────────────────────────────────────────────
```

### TP 실행 시 수수료 포함 수익률 확인

TP1~TP3는 수익이 충분할 때만 실행한다 (수수료를 내고도 이익이 나야 함):

```
수수료 구조:
  매수 시: 거래금액 × 0.015% (증권사 수수료)
  매도 시: 거래금액 × 0.015% + 거래금액 × 0.23% (수수료 + 증권거래세)
  왕복 총비용: 약 0.26%

최소 수익률 임계값 (시간 감소 적용):
  09:10 이전: 0.5% (수수료 0.26% + 슬리피지 여유분)
  시간이 지날수록 선형 감소
  15:15 이후: 0.1%
```

## 7. 시간 감소 수익률 임계값

오전에는 수익 기준을 높게 유지해서 성급한 청산을 막고,
장 마감 가까울수록 낮춰서 확실한 청산을 유도한다.

```
최소 수익률
  0.5% ━━━━━━━━━━━━┓
                    ┃
  0.4%              ┃
                    ┃
  0.3%               ┃
                     ┃
  0.2%                ┃
                       ┃
  0.1%                  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
       ──────────────────────────────────────────────────────────────────
        09:10         11:00         13:00         15:00  15:15

공식: threshold = 0.5% × (1 - 경과시간/(15:15 - 09:10))
     (09:10 이전 = 0.5%, 15:15 이후 = 0.1%)
```

## 8. 수수료 & 세금 구조

### 한국투자증권 수수료 체계

```
매수 수수료: 체결금액 × 0.015%
매도 수수료: 체결금액 × 0.015%
증권거래세: 체결금액 × 0.18% (KOSPI) / 0.18% (KOSDAQ)  
농어촌특별세: 체결금액 × 0.05%
─────────────────────────────────────
매도 총 부담: 약 0.245% (KOSPI 기준)
왕복 총 부담: 약 0.26%

예시: 10,000원짜리 100주 매수 후 10,150원에 전량 매도
  매수 수수료: 1,000,000 × 0.015% = 150원
  매도 수수료: 1,015,000 × 0.015% = 152원
  거래세:     1,015,000 × 0.23%  = 2,334원
  ────────────────────────────────────────
  총 비용:    2,636원
  실수익:     15,000 - 2,636 = 12,364원 (약 +1.24%)
```

## 9. 이메일 알림

09:20 스크리닝 완료 후 결과를 이메일로 발송한다.

### 이메일 구성

```
제목: [Stock Bot] 2026-04-14 스크리닝 결과: 5개 종목 감지

본문:
  - 스크리닝된 종목 목록 (표 형식)
    종목코드 | 종목명 | 갭(%) | 시총(억) | 거래대금(억) | 체결강도 | 상태

첨부파일:
```

```yaml

  bot:

    maxPositions: 5                   # 최대 동시 포지션 수

  screening:
    minGapPercent: 0.5                # 최소 갭 (%)
    maxGapPercent: 15.0               # 최대 갭 (%)
    minMarketCap: 50000000000         # 최소 시가총액 (500억원)

  scoring:
    enabled: true                     # false: 레거시 순차 필터 사용
    gapWeight: 30                     # 갭 가중치 (%)
    strengthWeight: 25                # 체결강도 가중치 (%)
    tradeValueWeight: 20              # 거래대금 가중치 (%)
    spreadWeight: 15                  # 스프레드 가중치 (%)
    marketCapWeight: 10               # 시가총액 가중치 (%)

  entry:
    highThresholdPercent: 1.5         # HIGH_FORMED 전환 조건 (시가 대비 +%)
    pullbackMinPercent: 1.5           # 최소 눌림목 깊이 (고점 대비 -%)
    pullbackMaxPercent: 3.0           # 최대 눌림목 깊이 (이 이상이면 FILTERED_OUT)
    bounceThresholdPercent: 0.3       # 반등 확인 기준 (저점 대비 +%)
    minPullbackMinutes: 3             # 눌림목 최소 지속 시간 (분)
    maxPullbackMinutes: 15            # 눌림목 최대 지속 시간 (분)

    sellTaxRate: 0.0023               # 매도세율 (0.23%)
    minProfitThreshold: 0.005         # TP 최소 수익률 (초반, 0.5%)

    timeDecayEnabled: true            # 시간 감소 수익률 임계값 활성화

  mail:
    enabled: false                    # true: 이메일 알림 발송
    to: "your-email@gmail.com"
```

## 11. FAQ & 트러블슈팅

### "09:20에 스크리닝 결과가 0개예요"

```
체크 사항:
□ stock.bot.enabled = true 인가?
□ KIS API 키가 올바르게 설정되었는가?
□ 오늘이 평일인가? (토/일 제외)
□ screening.minGapPercent 기준이 너무 높지 않은가?
□ API 토큰 발급에 실패했는지 로그 확인:
  grep "KIS auth" logs/stock-trading.log
```

### "체결강도가 항상 0으로 나와요"

KIS API는 장 초반에 체결강도를 집계하지 않아 0을 반환한다.
봇은 0을 "미집계"로 처리해 floor 필터에서 통과시킨다.
09:20 이후에는 정상 집계된다.

### "손절이 너무 자주 발동해요"

```
stopLossPercent: 1.5%는 매우 타이트한 손절이다.

완화 방법:
  1. stopLossPercent를 2.0~2.5%로 늘린다
  2. pullbackMaxPercent를 2.5%로 줄여 더 얕은 눌림목만 진입한다
  3. softEntryValidation을 false로 바꿔 진입 조건을 강화한다

주의: 손절을 넓히면 평균 손실도 커진다
```

### "공휴일에도 봇이 실행되려고 해요"

현재 봇은 공휴일 필터가 없다. 평일 cron만 있으므로
공휴일이 평일이면 스케줄이 실행된다.
수동으로 `stock.bot.enabled = false`로 설정하거나
봇을 종료해야 한다.

### "이메일이 오지 않아요"

```
체크 사항:
□ stock.mail.enabled = true 인가?
□ stock.mail.to 이메일 주소가 올바른가?
□ Gmail SMTP 설정이 되어 있는가?
  (spring.mail.host, spring.mail.username, spring.mail.password)
□ 로그에서 메일 오류 확인:
  grep "mail" logs/stock-trading.log
```
