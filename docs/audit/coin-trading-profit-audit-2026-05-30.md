## 1. 핵심 진단 (TL;DR)

- **출구 로직의 보상/위험 비율이 거꾸로다.** 승자는 +0.6%(net)에서 잘리고 패자는 −3%까지 흐른다(≈1:5). 손익분기 승률 **83.3%** 필요한데 1분봉 다지표 전략 현실 승률은 45~60% → **구조적 마이너스 기대값.** (`TradingProperties.java:169`, `application.yaml:153`, `TradingBotService.java:276-281`)
- **TP +15% / 트레일링 +10%는 사실상 죽은 파라미터다.** 1분봉 보유 윈도우에서 거의 발동 안 됨 → 모든 출구가 "+0.6% 승자 컷" 또는 "−3% 손절"로 붕괴. (`TradingProperties.java:165-167`)
- **리밸런싱이 Position을 만들지 않아 회계·리스크가 통째로 붕괴된다.** 리밸런스 매수 코인엔 stop-loss가 전혀 없고(리스크 루프는 OPEN Position만 순회), PnL/승률 지표에서도 누락. 손실 가드는 무포지션 시 sentinel +100%로 무력화, `min-sell-pnl-pct`는 yaml에서 −3%로 뒤집혀 ADR-0003 위반. (`RebalanceService.java:190-246, 299-305`, `application.yaml:167`)
- **안전장치가 없다.** PAPER/BACKTEST 모드도, 일일 손실 서킷브레이커도 없다 → 모든 파라미터 실험이 실거래이며 하락장에서 무한 출혈 가능.
- **SOL 시장 현실: 1분봉은 수수료에 진다.** SOL 1분 고저폭 ~0.1~0.25% < 왕복 수수료 0.5%. 일 ATR ~4%이므로 +15% TP는 3~4σ로 거의 미발생, −3% 손절은 ~0.75 ATR로 잡음에 너무 자주 걸림.

## 2. 수익을 갉아먹는 구조적 문제

### 2-1. 타임프레임 ↔ 수수료/엣지 불일치 (가장 근본적)
정상 매도는 수수료 차감 후 PnL ≥ 0.6%일 때만(`TradingBotService.java:276`) → 총가격 기준 약 **+1.6% 이동**(slippage 0.5% + 왕복 taker 0.5% + 마진 0.6%) 필요. 봇은 1분봉을 60초마다 재평가 → (a) 거짓 신호 평가 횟수 극대화, (b) RSI14/Stoch14/MA5를 미시구조 잡음으로 오염, (c) −3%/−2% 게이트를 잡음에 반복 트립 → **엣지 없는 회전·수수료 누수.**

### 2-2. 출구 비대칭 (단일 최대 누수)
정상 SELL은 PnL ≥ +0.6% net에서만 청산(승자 조기 컷), −0.6%~−3% 구간은 로그만 찍고 홀드(`:278-281`), 강한 SELL도 PnL ≥ −2%에서만(`:264-275`). 실현 분포 = **~+0.6% 승 vs ~−3% 패** → 70% 승률 가정해도 **EV = 0.70×0.6 − 0.30×3.0 = −0.48%/트레이드.** TP/트레일링이 죽어 승자를 키울 메커니즘이 없음.

### 2-3. Position ↔ 실잔고 회계 드리프트 (자본 보호 붕괴의 뿌리)
리밸런스 매수는 Trade만 저장하고 **Position 미생성**(`:190-207`), 리밸런스 매도는 OPEN Position을 안 건드림(`:232-246`). 파급:
1. **리스크 사각:** 불(70%) 구간 대부분 자본인 "유령 코인"에 **stop-loss/TP/트레일링 전무**(risk 루프는 OPEN Position만 순회 `:55`).
2. **손실 가드 무력화:** 무포지션 시 `calculateAveragePnlPct`가 **sentinel +100% 반환**(`:299-305`)해 가드 상시 통과.
3. **지표 오염:** ProfitService realizedPnl/승률/수수료가 CLOSED Position만 집계(`:85-112`) → 리밸런스 손익·수수료 **완전 누락.**
4. **정책 역전:** `min-sell-pnl-pct: -0.03`(`application.yaml:167`)이 ADR-0003(≥0%)·CLAUDE.md(+3%)를 무시하고 **−3%까지 손실 청산 허용.**

### 2-4. 신호 vs 리밸런스 엔진 충돌
두 엔진이 한 코인 잔고에 별도 쿨다운(신호 10분/리밸런스 8시간)으로 반대 방향 행동. `lastTradeTime`은 리밸런스가 갱신(`:197`)하지만 `lastRebalanceTime`은 신호가 갱신 안 해 **조정이 깨지고 틱 간 핑퐁** → 매 레그 0.25% taker.

## 4. 대상 페어 = ADA 확정 + ADA 결합도 점검

> 사용자 확정(2026-05-30): **대상 페어는 KRW-ADA**. (이전 "SOL" 언급은 착오.)

### 4-2. 시사점
- **ADA 특성이 *전략 판단* 에 숨어 들어간 곳은 없다.** 로직은 페어 무관, ADA는 (a) 튜닝된 값 + (b) 스냅샷 스키마/기본값의 이름표로만 존재.
- 타깃이 ADA로 고정되었으므로 **§3 P1 의 손절/익절/트레일링 재보정은 ADA 변동성 기준**으로 진행(SOL 표 폐기). MA-convergence 억제·`minAgreeingIndicators=3`은 ADA 횡보장 휩쏘 방지에 유효 → **유지.**
- 결합 항목 중 기능 영향이 있는 것은 없음. `wslKRW-ADA` 오타와 `setScale(0)` 가정만 위생/메모 차원.

## 6. 반박·정정된 주장 (오해 방지)
- **"limit 주문으로 수수료 절반 절감" — Bithumb에선 틀림(정정).** maker=taker=0.25% 동일. limit는 슬리피지만 절감, 진짜 수수료 레버는 0.04% 쿠폰.
- **RSI "~2배 노이지" / "1분봉이 단일봉에 1.1% 못 만들어 손실"** — 코드 사실(비-Wilder, 1분 fetch)은 맞으나 배수·메커니즘은 미검증 이론(critical/high→**medium 강등**). Wilder 전환 권고 자체는 타당.
- **모멘텀 vs 평균회귀 "~+5점 순상쇄"** — illustrative(high→**medium**). 더 강한 근거는 below-MA60 확인 게이트(`SignalService.java:296-311`).
- **신호↔리밸런스 "같은 틱 동시"** — **반박됨**(`:198` 조기 return). 핑퐁은 틱 간 발생(핵심 논지 유효).
- **거래량 다이버전스 ±20 "과대가중"** — 가중치 사실은 확정이나 "약한 예측자" 해악은 백테스트 없는 prior(medium→**low**, 튜닝 제안).
- **stop-loss 주석 "−8%" / `SignalScoringService.java` 오참조 / market 오타** — 모두 **low(문서·위생)**, 런타임 무영향.

---

**결론:** 적자는 단일 버그가 아니라 **(1) 출구 비대칭 + (2) 타임프레임-수수료 불일치 + (3) Position↔잔고 회계 붕괴(+리밸런스 손실청산)의 합성**. 순서는 **P0 안전장치 → P1을 백테스트로 검증해 적용 → 페어/타임프레임 변경 시 % 임계 전부 ATR 배수로 재보정.**
