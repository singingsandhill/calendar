# ADR-0005 (trading/strategy): 출구 R:R 재보정 — 손절/익절/트레일링 (ADA)

| 항목 | 값 |
|---|---|
| 상태 | Accepted |
| 날짜 | 2026-05-30 |
| 도메인 | trading (코인) |
| 관심사 | 알고리즘 / 리스크 |
| 관련 | 수익성 감사 `docs/audit/coin-trading-profit-audit-2026-05-30.md` (P1-1/P1-2) |

## Context — 무엇이 문제였나

감사 결과, 출구 파라미터가 **보상/위험을 거꾸로** 만들고 있었다.

- 손절 −3%, 익절 +15%, 트레일링 활성 +10%/추적 −3%.
- 1분봉 알트(ADA) 보유 윈도우에서 **+15% TP·+10% 트레일링은 사실상 발동 안 함**(죽은
  파라미터) → 모든 출구가 "신호 매도(작은 이익)" 또는 "−3% 손절"로 붕괴.
- 실현 분포가 *작은 승 vs −3% 패* → 손익분기 승률 ~83% 필요(구조적 마이너스 기대값).

## Decision — 무엇을 골랐나

ADA 기준 권장값으로 재보정(전부 `risk.*` 설정값 — 코드 로직 불변).

| 파라미터 | 이전 | 변경 | 의도 |
|---|---|---|---|
| `stop-loss` | −3% | **−1.5%** | TP +3% 와 **1:2 R:R** (손익분기 승률 83%→33%) |
| `take-profit` | +15% | **+3%** | 1분봉 알트 인트라데이에서 실현 가능 |
| `trailing-activation` | +10% | **+1.5%** | 의미있는 이익에서 트레일링 활성 |
| `trailing-stop` | −3% | **−0.8%** | 트레일링을 **주 승자 출구** 로 (손익분기 floor 유지) |

권장 범위 중 *보수적 끝*(넓은 손절/넓은 트레일 = 잡음 청산 ↓) 선택. 모든 값은 yaml
설정으로 튜닝 가능.

## Rationale — 왜 이 선택인가

- TP/트레일링이 "죽은" 원인은 임계가 ADA 의 1분봉 이동폭 대비 과대(3~4σ)였기 때문.
  실현 가능한 범위로 낮춰 트레일링이 실제 승자 출구가 되게 함.
- 손절을 −1.5% 로 좁히되 TP +3% 와 1:2 를 만들어 낮은 승률에서도 +EV 가능.
- 대안(ATR 배수 동적 임계)은 더 견고하나 추가 구현 — 우선 고정 권장값으로 시작,
  백테스트 후 ATR 화 검토.

## Consequences — 영향

- **긍정:** 보상/위험 1:5(역) → **1:2**, 손익분기 승률 83%→~33%. 트레일링이 승자 포착.
- **위험/주의:** 좁은 손절(−1.5%)은 1분봉 잡음에 더 자주 트립 가능(회전 ↑). **기본
  LIVE 이므로 PAPER 백테스트로 +EV·트립 빈도 확인 후 확정 강력 권장** (`TRADING_BOT_MODE=PAPER`).
- `strongSignalMaxLoss(−2%)` 는 손절(−1.5%)이 먼저 발동해 실질 무영향(허용 가드라 무해).
- 후속: ATR 배수 동적 임계(MA-convergence 도 ATR% 상대화) 검토 — 페어/변동성 변화 대응.
