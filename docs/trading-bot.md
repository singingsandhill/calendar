# 코인 트레이딩 봇 — Bithumb ADA/KRW

> **대상 독자:** 봇을 운영하거나 코드를 유지보수하는 사람  
> **목적:** "봇이 왜 이렇게 작동하는가"를 이해하기 위한 레퍼런스

---

## 1. 전략 개요

### 핵심 철학: 다중 지표 합의

이 봇은 **어떤 하나의 지표를 믿지 않는다.** 8가지 독립적인 지표가 같은 방향을 가리킬 때만 거래한다.

```
"하나의 지표는 거짓말할 수 있다. 여덟 개가 동시에 거짓말하기는 어렵다."
```

MA 크로스가 매수 신호를 내더라도 RSI가 과매수이고 거래량 다이버전스가 하락을 가리키면 거래하지 않는다.
→ **노이즈에 의한 잘못된 거래(휩소)를 줄이는 것이 최우선 목표다.**

### 봇이 잘 작동하는 조건

- 트렌드가 명확한 시장 (강한 상승 또는 하락)
- 거래량이 충분한 시장 (다이버전스 신호의 신뢰도 확보)
- 변동성이 적당한 구간 (ATR 1~3%)

### 봇이 불리한 조건

- 횡보 장세: MA가 수렴하면 크로스 신호 억제됨 (MA 수렴 억제 규칙)
- 거래량 급감 구간: 다이버전스 신뢰도 저하
- 극단적 변동성: ATR > 3%면 주문 규모를 15%로 줄여 리스크 제한

## 4. 기술적 지표 계산

### 이동평균선 (Simple Moving Average)

```
MA(n) = (최근 n개 종가의 합) / n

MA5  = 5분 평균  → 단기 모멘텀
MA20 = 20분 평균 → 중기 추세
MA60 = 60분 평균 → 장기 추세 기준선 (매수/매도 판단의 핵심)
```

### RSI (Relative Strength Index, 기간 14)

```
평균 상승 = 최근 14개 캔들 중 상승폭 평균
평균 하락 = 최근 14개 캔들 중 하락폭 평균
RS  = 평균 상승 / 평균 하락
RSI = 100 - (100 / (1 + RS))

기준값:
  RSI < 35 → 과매도 (매수 신호)
  RSI > 65 → 과매수 (매도 신호)
```

### 스토캐스틱 오실레이터 (기간 K=14, D=3)

```
%K = (현재 종가 - 최근 14개 중 최저가) / (최근 14개 중 최고가 - 최저가) × 100
%D = %K의 3기간 이동평균

기준값:
  %K < 25 → 과매도 (매수 신호)
  %K > 75 → 과매수 (매도 신호)
```

### ATR (Average True Range, 기간 14)

```
TR(진정범위) = max(고가-저가, |고가-전일종가|, |저가-전일종가|)
ATR = TR의 14기간 이동평균
ATR% = ATR / 현재가 × 100

용도: 변동성 측정 → 주문 규모 결정 (섹션 7 참고)
```

### 거래량 이동평균

```
VolumeMA = 최근 20개 캔들 거래량의 평균
용도: 거래량 다이버전스 탐지 기준
```

## 5. 8지표 합성 신호 시스템 ★ 핵심

각 지표는 독립적으로 점수를 부여하고, 합산 점수로 매매를 결정한다.

```
┌────────────────────────────┬──────────┬──────────┬────────────────────────────────────┐
│ 지표                       │ 매수점수 │ 매도점수 │ 발동 조건                          │
├────────────────────────────┼──────────┼──────────┼────────────────────────────────────┤
│ MA 크로스 — 이벤트         │  +25     │  -25     │ 이번 캔들에서 골든/데드크로스 발생 │
│ MA 크로스 — 상태유지       │  +10     │  -10     │ MA5 > MA20 (또는 < MA20) 유지 중   │
│   ★ MA 수렴 억제           │    0     │    0     │ |MA5-MA20| / MA20 < 0.2% 이면 무시│
│ MA 추세                    │  +15     │  -15     │ 현재가가 MA60 위(또는 아래)        │
│ RSI 다이버전스             │  +20     │  -20     │ 가격은 저점 갱신, RSI는 저점 미갱신│
│ RSI 레벨                   │  +15     │  -15     │ RSI < 35 (과매도) / RSI > 65 (과매수) │
│ Stoch 다이버전스           │  +15     │  -15     │ 가격과 %K의 고점/저점 불일치       │
│ Stoch 레벨                 │  +15     │  -15     │ %K < 25 (과매도) / %K > 75 (과매수) │
│ Volume 다이버전스          │  +20     │  -20     │ 가격 저점 갱신, 거래량 저점 미갱신 │
│ RSI 추세                   │  +10     │  -10     │ RSI가 상승 추세 / 하락 추세        │
├────────────────────────────┼──────────┼──────────┼────────────────────────────────────┤
│ 이론적 최대 합산           │  +135    │  -135    │ 단, MA크로스는 이벤트/상태 중 하나 │
│ 실질적 최대 합산           │  +125    │  -125    │ (이벤트 +25 OR 상태유지 +10)       │
│ 매매 임계값                │  ≥ +40   │  ≤ -40   │ + 아래 추가 필터 조건              │
└────────────────────────────┴──────────┴──────────┴────────────────────────────────────┘
```

### MA 수렴 억제 — 왜 필요한가?

```
상황: MA5와 MA20이 거의 같아서 위아래로 반복 교차하는 횡보 구간

문제: 크로스가 계속 발생 → 매수했다가 바로 매도 신호 → 수수료만 낭비

해결: |MA5 - MA20| / MA20 < 0.2% 이면 크로스 점수를 0으로 만든다
     → 두 선이 실질적으로 같은 위치에 있을 때는 신호를 무시
```

### 다이버전스(Divergence) 탐지 원리

```
상승 다이버전스 (매수 신호):
  가격: 저점 A → 더 낮은 저점 B (하락)
  지표: 저점 A → 더 높은 저점 B' (상승)
  의미: 가격은 계속 하락하지만 지표의 하락 압력이 약해짐 → 반등 임박

하락 다이버전스 (매도 신호):
  가격: 고점 A → 더 높은 고점 B (상승)
  지표: 고점 A → 더 낮은 고점 B' (하락)
  의미: 가격은 계속 상승하지만 지표의 상승 모멘텀이 약해짐 → 하락 임박

탐지 방법: 최근 20개 캔들에서 극값(피크, 골짜기)을 찾아 비교
```

## 6. 매매 결정 로직

### 매수(BUY) 조건 — 모두 충족해야 함

```
① 합산 점수 ≥ +40
② RSI < 70 (과매수 구간 진입 전)
③ Stoch %K < 85 (과매수 극단 회피)
④ 최소 3개 이상의 지표가 매수 방향으로 동의

⑤ 가격이 MA60 아래인 경우 — 추가 확인 필요 (하락 추세에서의 매수이므로)
   다음 중 하나 이상 충족해야 함:
     - 상승 다이버전스 탐지
     - RSI < 30 (강한 과매도)
     - 현재 거래량 > VolumeMA × 1.5 (거래량 급증)
```

### 매도(SELL) 조건 — 모두 충족해야 함

```
① 합산 점수 ≤ -40
② RSI > 30 (과매도 구간 진입 전)
③ Stoch %K > 15 (과매도 극단 회피)
④ 최소 3개 이상의 지표가 매도 방향으로 동의

일반 매도: 현재 수익률 ≥ 0.6% (수수료 0.5% 포함 최소 수익)
강한 매도 (score ≤ -60 또는 하락 다이버전스):
          현재 손실 ≥ -2% 이면 즉시 청산 (추가 손실 방지)
```

### 휩소(Whipsaw) 방지 장치

```
매매 쿨다운:   마지막 거래 후 10분 이내 → 새 거래 금지
최소 보유 시간: 매수 후 15분 이내 → 매도 금지
MA 수렴 억제:  두 이동평균이 거의 같은 위치 → 크로스 신호 무시
지표 합의 최소: 3개 이상 동의 없으면 → HOLD
```

## 7. 포지션 사이징 — ATR 기반 동적 주문

변동성이 클수록 위험하므로 주문 규모를 줄인다.

```
ATR%    주문 비율    계산 예시 (KRW 잔고 1,000,000원)
────────────────────────────────────────────────────
≥ 3.0%    15%      → 150,000원 주문 (고변동성, 보수적)
  2.5%    20%      → 200,000원 (선형 보간)
  2.0%    25%      → 250,000원 (기본 수준)
  1.5%    30%      → 300,000원
≤ 1.0%    35%      → 350,000원 (저변동성, 적극적)
────────────────────────────────────────────────────

실제 주문금액 = KRW잔고 × 비율 × (1 - 0.5% 슬리피지 버퍼)

예시: ATR% = 2.0%, KRW = 1,000,000원
    → 1,000,000 × 25% × 0.995 = 248,750원
```

> **왜 ATR을 쓰는가?** 변동성이 큰 날은 같은 손절 기준(-3%)으로도 훨씬 빠르게 청산될 수 있다.
> ATR로 주문 크기를 줄이면 실질 리스크(손실 금액)를 일정하게 유지할 수 있다.

## 8. 리스크 관리 규칙

매분 거래 루프에서 **오픈 포지션마다** 다음 6단계를 순서대로 실행한다:

```
[1단계] 손절 체크
    수익률 ≤ -3% → 즉시 전량 청산 (이유: STOP_LOSS)
    └─ 더 이상 기다리지 않고 손실 확정

[2단계] 고점(High Water Mark) 갱신
    현재가 > 지금까지의 최고가 → 최고가 업데이트
    └─ 추적손절의 기준이 되는 값

[3단계] 추적손절 활성화 체크
    수익률 ≥ +10% 이고, 추적손절이 아직 비활성화 상태이면:
    └─ 추적손절 시작가 = 현재가 × 0.97 (3% 아래)

[4단계] 추적손절 갱신 & 청산 체크
    추적손절 작동 중이면:
      추적손절가 = max(현재 추적손절가, 최고가 × 0.97)
      안전장치: 추적손절가 ≥ 진입가 × (1 + 0.5%) (손해 보고 추적손절되지 않도록)
      현재가 ≤ 추적손절가 → 즉시 전량 청산 (이유: TRAILING_STOP)

[5단계] 익절 체크
    수익률 ≥ +15% → 즉시 전량 청산 (이유: TAKE_PROFIT)

[6단계] 상태 저장
    추적손절 상태(고점, 추적손절가) DB에 저장
```

### 추적손절 시나리오 예시

```
진입가: 1,000원

시간 흐름:
─────────────────────────────────────────────────────────────────
 +5%  → 1,050원: 추적손절 아직 비활성화
 +10% → 1,100원: 추적손절 활성화!
                  추적손절가 = 1,100 × 0.97 = 1,067원

 +13% → 1,130원: 최고가 갱신
                  추적손절가 = 1,130 × 0.97 = 1,096.1원

 +15% → 1,150원: 익절(+15%)로 먼저 청산 OR
 (또는)
 하락 → 1,095원: 추적손절가(1,096원) 이하 → 추적손절 청산
                  결과: 약 +9.5% 수익 확정 (최고점 대비 -3%)
─────────────────────────────────────────────────────────────────
```

### 수수료 포함 수익률 계산

```
미실현 손익 = (현재가 × 수량) - 진입금액 - (매수수수료 + 예상 매도수수료)
수익률     = 미실현 손익 / 진입금액 × 100

Bithumb 수수료 (taker): 0.25%
최소 수익 임계값: 0.6% (수수료 0.5% + 슬리피지 0.1% 커버)
```

## 9. 포트폴리오 리밸런싱

### 목적

매매 신호가 없더라도 **시장 추세에 맞게 코인/KRW 비율을 유지**한다.

### 목표 비율

```
시장 상태 판단 기준: 현재가 vs MA60 (60분 이동평균)

현재가 > MA60 (상승 추세) → 코인 70% / KRW 30% (공격적 보유)
현재가 < MA60 (하락 추세) → 코인 30% / KRW 70% (방어적 현금화)
MA60 데이터 부족          → 리밸런싱 건너뜀 (skipWhenDataInsufficient)
```

### 리밸런싱 실행 조건

```
① 기능 활성화 (rebalancing.enabled = true)
② 마지막 리밸런싱 후 8시간 경과
③ 현재 비율과 목표 비율의 차이 ≥ 10%
④ 최소 주문금액 5,000원 이상

매도 리밸런싱 추가 조건:
   오픈 포지션의 평균 수익률 ≥ +3% (손해 보는 포지션을 억지로 팔지 않음)
```

### 강한 신호 우선순위

```
합산 점수 ≥ +60 (강한 매수): 리밸런싱을 건너뛰고 즉시 매수 실행
합산 점수 ≤ -60 (강한 매도): 리밸런싱을 건너뛰고 즉시 매도 실행

이유: 강한 모멘텀이 있을 때 리밸런싱 주문이 기회를 놓치지 않도록
```

              │
              ▼
     [신호 생성: 8지표 점수 계산]

## 12. FAQ & 트러블슈팅

### "왜 매수가 안 되나요?" 진단 체크리스트

```
□ trading.bot.enabled = true 인가?
□ KRW 잔고가 5,000원 이상인가?
□ 현재 오픈 포지션이 maxPositions(2) 미만인가?
□ 마지막 거래 후 10분 이상 지났는가? (쿨다운)
□ 합산 점수가 +40 이상인가? (로그에서 확인)
□ RSI가 70 미만인가?
□ Stoch %K가 85 미만인가?
□ 3개 이상의 지표가 매수 방향인가?
□ 가격이 MA60 아래라면 — 다이버전스/RSI<30/거래량 급증 중 하나 있는가?
```

### "추적손절이 손해에서 발동했어요"

추적손절의 안전장치가 작동했는지 확인한다:
- 안전장치: `추적손절가 ≥ 진입가 × 1.005` (진입가 + 수수료)
- 이 조건으로 인해 수익이 거의 없는 상태에서도 추적손절이 발동할 수 있다

### "리밸런싱이 자주 발동하지 않아요"

```
체크 사항:
① rebalancing.enabled = true?
② 마지막 리밸런싱 후 8시간 지났는가?
③ 현재 코인/KRW 비율과 목표 비율 차이가 10% 이상인가?
④ MA60 데이터가 충분히 쌓였는가? (캔들 60개 이상)
```
